Many attributes like stroke and fill are inherited from a parent, except attributed that define the core geometry of an element.

Many styles are inherited from parent to children. Geometry attributes are usually not inherited.

Predefined elements can be used with the defs and use keywords.Define an element in the defs body and give it an id attribute. It will not be rendered. An instance is created with <use href="#elementId" />. The referenced element doesn't have to be defined in the defs body, any element will do.

You may have to namespace-prefix the href attribute for older hosts:

Style Sheets

An SVG element may contain CSS style elements. They are usually kept near the top, before or after the defs element

Inline SVG can be styled with external stylesheets.

Text

Text is created with text elements.The text-anchor attribute defines the reference ooibt for positioning. Possible values are left, right and center.

Text colors are set with stroke and fill attributes. The stroke color is usually set to none, although it can be used for an outline effect. Note that the CSS attribute color doesn't work.

Radial Gradients

The radialGradient element creates (you might have guessed it) a radial gradient. Just put a number of stop color elements in the body. fx and fy attributes define the center of the effect.

Linear Gradients

Put a number of stop color elements in the body to define the effect.

SMIL Animations

Are not suported by Microsoft, and therefore other browser developers don't much effort into SMIL.

CSS Animations

CSS animations are not a full replacement for SMIL, but better supprted by browser manufacturers. Presentation attributes can be overridden by CSS rules. Inline CSS cannot be overridden by stylesheet rules. No all CSS attributes can be animated.
